About This Project

Purpose

This proof of concept demonstrates the integration of multiple AWS services into a cohesive, production-ready architecture. It showcases industry best practices for deploying web applications on AWS with proper security, scalability, and maintainability considerations.

The project is designed as a learning resource and template for teams looking to understand how these AWS services work together in real-world scenarios.

Technology Stack

AWS CDK (TypeScript)

Infrastructure as Code

FastAPI (Python)

Modern async API framework

Gunicorn + Uvicorn

Production ASGI server

GitHub Actions

CI/CD Pipeline

Security Features

  • WAF Rules: Protection against OWASP Top 10 vulnerabilities
  • Rate Limiting: Prevent abuse and DDoS attempts
  • Origin Access Control: S3 bucket is private, accessible only via CloudFront
  • Security Groups: Minimal ingress rules, EC2 only accepts traffic from ALB
  • IAM Roles: Least privilege principle for all resources
  • Encryption: Data encrypted at rest (S3, EBS) and in transit (HTTPS)

Deployment

The infrastructure is deployed using AWS CDK with a single command:

# Deploy the entire stack npm run cdk deploy # Or deploy with approval npm run cdk deploy --require-approval broadening

CI/CD is configured via GitHub Actions, automatically deploying changes when code is pushed to the main branch.

Cost Optimization

  • t4g.nano: ARM-based Graviton processor for best price/performance
  • CloudFront Caching: Reduces origin requests
  • Single AZ: For PoC purposes (use Multi-AZ in production)
  • No NAT Gateway: EC2 in public subnet for cost savings in PoC
